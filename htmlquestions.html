1.What is HTML?

HyperText Markup Language: standard markup language for web pages.

2. Why do we use HTML?

To structure and organize content on the web.

3. What is Doctype in HTML?

Declaration of document type, specifying HTML version (e.g., !DOCTYPE html).

4. What is the root element of HTML? - element.

html element.

5. What is a tag?

A keyword surrounded by angle brackets (e.g., p tag).

6. Examples of self-closing tags?

img tag, br tag, hr tag, input tag, link tag.

7. What is an element?

A tag with content
Opening tag + Content + Closing tag

8. Types of elements in HTML?

Two main types: inline and block.

9. Difference between inline vs block elements?

Inline: display: inline (e.g., span tag, anchor tag), occupies only necessary space.
Block: display: block (e.g., paragraph tag, div tag), occupies full width

10.Difference between semantic elements and non-semantic elements?

Semantic: convey meaning (e.g., header tag, footer tag).
Non-semantic: no inherent meaning (e.g., div tag, span tag).

11.What are href vs src attributes?

href: links to external resource (e.g., a href="url" tag).
src : embeds external resource (e.g., img src="image.jpg" tag).

12.Attributes for img tags?

src, alt, width, height, title.

22.What is git vs github?

23.What is the workflow of github?

1. Create repository
2. Clone repository
3. Make changes
4. Stage changes (git add)
5. Commit changes (git commit)
6. Push changes (git push)
7. Pull changes (git pull)

24.What command do we need to use to move the changes to staged changes?

Move changes to staged changes: git add file-name

25. How can you stage all files at a time?

27.What is git commit command usage?

Git commit command: git commit -m "commit-message"
